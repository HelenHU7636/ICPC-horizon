# 动态规划总结 · 按"状态设计"而非"题型外壳"分类

## 为什么这样总结

不按 "01 背包 / 完全背包 / 多重背包" 这种**外壳**分类——那是死模板,换个皮就失效,考试不会原样出。

也不只按 "求最优值 / 求方案数 / 判断可行" 分类——这个分类是对的,但它只描述了 **cell 里存什么**,没描述**状态怎么定义**。而难题真正难的地方,恰恰在后者。

> 例:LC416 子集和。一开始就知道它"求可行性",但仍然卡住——卡点不在"存什么",而在"怎么给状态编号"。所以只靠值域分类救不了场。

正确的切法是**两个正交的轴**。

---

## 思考维度

状态轴:状态按什么编号——输入位置 / 输出值 / 两者都要。决定表的形状。
目标轴:你在算什么——最大最小(最优化)/ 方案数(计数)/ 能否达到(可行性)/ 期望。决定合并算子是 max、min、加号还是"或"。
依赖轴:转移依赖什么——点对点(dp[i] 只看相邻 dp[i-1])/ 一段区间(区间 DP)/ 一个子集(状压)。决定复杂度,也决定要不要上单调队列这类优化。
顺序轴:顺序算不算数——这就是你们 377 和 518 的区别,排列还是组合,直接决定循环谁在外层。
决策轴:每步在决定什么——选/不选(背包)/ 多选一 / 在哪儿切一刀(区间 DP 枚举分割点)。

妙处在于:一道题沿这五条轴定完位,表几维、base case 是什么、循环往哪个方向走,基本就被推出来了,而不是回忆出来的。没见过的题也变成"已知轴的新组合",而不是"未知题型"。这套轴本身就是你们这个模块最该沉淀的东西。

## 该背在哪一层

模板和第一性思考不是对立的,是**分层**的。象棋高手背定式背得比谁都熟,但背的是定式背后的**为什么**;到了没有定式的中局,调用的是从定式里抽出来的**原则**。问题不是"背不背",是"背在哪一层"。

- **第一层 · 死模板**(没用):背 "01 背包就是倒序两重循环"。脆,换皮失效。
- **第二层 · 触发条件 + 状态设计**(金子,性价比最高):背 "看到'从一堆东西选、凑目标值',状态就按目标值编号,而不是按选了哪些编号"。
- **第三层 · 第一性推导**(最该保护的能力):从问题定义现场推出递推式(如自己逆推出倒序遍历的必要性)。

只总结 "最终求什么 + 常在哪出错",漏掉的正是**第二层**。而第二层最划算:不像第一层那样脆,也不像第三层那样每题从零推、考场没时间。难题的本质常常是"用一个见过的**状态设计套路**,去套一个没见过的**问题外壳**"。

---

## 第二层的核心:问题信号 → 状态设计 映射表

不是模板代码,而是**条件反射**:什么信号触发什么状态设计。

| 看到这种信号 | 状态设计 | 细节 |
|---|---|---|
| 两个序列怎么对应 / 比较 | 按两个输入位置编号,二维表 | 点对点,读相邻几格 |
| 从一堆数里选,凑出某个目标值 / 容量 | 按输出值(目标)编号 | 每个元素作用在全表 = 全盘刷新 |
| 每个元素选或不选,只能用一次 | 0/1 | **倒序**刷新和轴 |
| 元素可重复使用 | 完全背包 | **正序**刷新和轴 |
| 在序列上做选择,当前只依赖前面某位置的选择 | 按位置编号 + 在前缀里找最优 | LIS 一类 |

> 扫描方向记忆锚:每个数只能用一次 → 影响只往大处传 → **从大往小写**才不会同一轮踩到自己刚写的结果;可重复用 → **就是要**复用同轮刚更新的格子 → 正序。

---

## 考场上的真实流程

```
看到新题
  → 外壳没见过(正常,难题/ICPC 不会出原样模板)
  → 但识别出"从一堆东西凑目标值"这类【信号】(第二层)
  → 条件反射:按输出编号
  → 剩下细节(存什么、扫描方向)用【现场推导】补上(第三层)
```

这是模板与第一性思考的**合作**,不是二选一。

---

## 一句话结论

不用赌考试偏模板还是偏新题:

- 偏模板 → 识别信号后直接秒。
- 偏新题 → 定位方向的速度远快于从零推,省下的时间留给真正需要推导的硬骨头。

**第二层(信号 → 状态设计)在两种情况下都赢。**
