# 解析本题的几重复杂度
1. permutation-n个元素集合的可能permutation O(n!)
2. 普通dfs求一个数组所有子数组，O(2^n)
3. i,j，给定一个集合，遍历其所有子区间，O(n^2)

在一个问题中，所需信息的总数是守恒的。数据存放的有序性能提供一部分信息，搜索本身也能直接提供信息。如果数据存放完全无序，所有信息都要通过搜索获取，那么时间复杂度就会很大。
纯粹搜索会带来巨大的时间复杂度，这提示我们，必须观察skyline表达式的规律，排除一些必须搜索的路径，同时恰当排列数据，以减少搜索的需要。

以下是对skyline表达式的分析：

由“最小值”*“长度”两部分组成，正常人此时的想法是，要想让skyline最大，要让最小值最大同时长度最大。

但是，二者此消彼长，长度越大，在这么长的子区间里包含进较小元素的可能性就越大，从而导致最小值变小；反之，若长度为1，最小值可以取到数组中的最大元素。

此时，应当停下来，去观察其中一方的变化规律。

**当长度固定时，有没有一种排列，能让所有子区间平均最小值最大？**

例如，长度为2的子区间，{100,1,200},此时子区间{100,1}的最小值为1;
{100,200,1}，子区间{100,200}的最小值为100.
可以看到，把大数和大数放在一起，就能让大数组的最小值变大，因此排序后的数组所有子区间平均最小值最大

# 排序使得O(n!)的permutation坍缩成了唯一一种有序情况
现在，问题已经从：
在每一种全排列中遍历所有子区间，使得子区间的skyline最大。
变成：
**在有序情况这一种排列中遍历所有子区间，使得子区间的skyline最大**

# 下一主要复杂度来自“所有”子区间可能性太多

**能否筛选出部分子区间，使得我们最后要选的那一个子区间就在其中？也就是说，能否缩小子区间候选人的范围？**

再次观察“长度*最小值”这个式子，显然，长度固定时，区间越靠近有序数组更大的一端，最小值最大。

但是，长度变化时，长度大最小值小，与长度小最小值大，仍然是一个不可调和的博弈，例如：
{1,2,3,1000000}
1*1000000
4*1
此时选长度小最小值大的
但
{1,2,3,4,5}
1*5
2*4
此时选长度大最小值小的
能否找到一个规律，何时取长度，何时取值？
不能，这个结果并非单调，不同长度下，二者优先级不同，所以要想判断哪个占优，必须先确定长度
但长度不固定，
比如，
长度为2时，2*4可能占优势，但长度为5时，1\*5又不再占优势
因此，必须遍历每种可能的长度与最小值组合。
但此时由于数组的有序性，长度给定时最小值的取值具有相当明显的规律，
以数组降序排列为例，
1->max-1st-idx=0
2->min-2nd-idx=1

i->[i-1]

此时复杂度已经由O(n^2)降低到O(n)

# f(X)+f(Y)

同样的道理，正常人一定能想到，要想让一个和最大，最好能让两个加数分别取到最大值
但是，X与Y并不独立，其值依赖于数组的划分方式，当一个确定时，另一个也随之确定，不存在独立变化的可能。
因此，这个“和”必须当做一个整体处理，且f(y)可以写成f(x)的函数
f（x）的所有可能取值已经由第二部分解决，只需确定每一个f（x）对应的f（y），也就是刨除x后原数组剩余部分的skyline

原数组剩余部分，仍然是一个数组，所以又可以用同样的方法求得f

遍历所有可能的xy组合，筛选其和的最大值。
