# 线性 DP · 一维递推型 复习笔记

---

## 它是什么

**线性 DP**：所有前驱都在这个一维数组里、位于当前下标**之前**。沿输入序列的下标线性排开，推进方向单一。

---

## 核心动作：拷问最后一步

> **"要到达状态 i，最后一个动作有哪些可能？"**

把这些可能**不重不漏**地列出来，每种都把问题缩小成一个更小的同类子问题——递推式就自己冒出来了。

站在状态 i，这一问会自动逼出三个环节，按顺序问：

### ① 这一步能从哪些前驱来？——**筛前驱**

约束作用在"谁有资格当前驱"上，分两类，可能叠加：

- **位置型约束**（相邻 / 连续 / 前 k / 固定偏移）：**圈定范围**，把"前面所有"缩成"某几个"。
  - 相邻 → 圈成 `i-2`
  - 连续 → 圈成 `i-1`
  - 前 k → 圈成窗口 `[i-k, i-1]`
- **值型约束**（递增 / 整除 / 差固定）：在范围内**再筛**，`nums[j]` 跟 `nums[i]` 比一下，合格的才能接。

> 两者都是在**缩小前驱集合**，只是判断标准不同；但**最终都落到下标 j 上**——因为数组终究是用索引筛的。

### ② 多个合法前驱，挑一个还是全要？——**合并方式**

- **挑**（带决策）→ `max` / `min` / `if-else`。从几条路里选最优。
- **全要**（无决策）→ `+`。几条路都算数，累加。


### ③ 判"能不能走这一步"时，现有状态够吗？——**要不要升维**

若"能否走这一步"取决于一个 **`nums[j]` 查不出来的隐藏量**，状态就不够描述"最后一步"，补一维。

- 摆动序列 → 补**方向**（上升/下降）
- 等差子序列 → 补**公差**
- 不能连抢三家 → 补**连抢计数**

> **判据：这个信息 `nums[j]` 查得出吗？** 查得出 → 它只是个筛选条件，不占维度；查不出（历史状态）→ 才升一维。升维不是额外规则，是"拷问最后一步时发现信息不够"的补救。

---

## 底层铁律

> **dp 下标永远是位置；值靠 `nums[下标]` 现查——别拿值当下标。**

位置（下标 j）、值（`nums[j]`）、子问题解（`dp[j]`）三样通过同一个下标 j 全串起来，一个都不用塞进索引里。
正因如此：位置约束只圈范围（下标的事），值约束只当筛子（`nums` 现查的事），升维只发生在"现查不到"的时候。

---

> 附：状态若锚了"以 i 结尾"，真答案不一定结在 i，最后要在 `dp` 上横扫一遍 `max`。这是**状态定义的副作用**，定义时当场就知道，不单独记。

**边界易错点：**

1. 初值给够（回看 i-2 要 2 个初值，i-3 要 3 个）
2. 特殊情况如input vector为空。

### 通用骨架——一维递推、滚动变量、O(1) 空间

```cpp
// 一维递推型线性DP 通用骨架（滚动变量版）
// 适用：dp[i] 只依赖前面固定常数个状态
long long solve(int n) {
    long long a = /*dp[0]*/, b = /*dp[1]*/;   // 初值给够（窗口=2 就 2 个）
    if (n == 0) return a;
    if (n == 1) return b;
    for (int i = 2; i <= n; i++) {
        long long cur = /* f(a, b)，即 dp[i] 的递推式 */;
        a = b; b = cur;                        // 窗口整体右移一格
    }
    return b;
}
```
