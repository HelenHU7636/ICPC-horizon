# 背包 / 凑数类 —— 复习笔记与板子库

> 适用范围：0-1 背包、完全背包、子集和、凑零钱（最少 + 方案数）、目标和。
> 这一类清单里被标了一句话：**最容易写错的是循环顺序**。本笔记的重点就是把"为什么这么写"讲清楚，而不是让你背规则。

---

## 0. 先认出这一类：它的"签名"

你之前所有 DP（线性、双序列、网格）的 `dp` 下标都是**位置**——第几个元素、第几行第几列。背包凑数类不一样：

> **它的 `dp[j]`，下标 `j` 是"你要凑出来的那个值"**（容量 / 金额 / 和 / 目标数），不是位置。

这就是清单里标的 **output-directed**：状态从"挂在输入位置上"挪到了"挂在输出目标上"。

具体感受一下区别：

- 线性 DP（如 LIS）：`dp[i]` = 以第 i 个元素结尾的最长上升子序列长度。下标 `i` 是位置。
- 背包（如凑零钱）：`dp[j]` = 凑出金额 `j` 最少要几枚硬币。下标 `j` 是金额，是你想要的结果。

**为什么你会觉得这一类别扭、和子集和/凑零钱"看不出是一类题"？**
就是因为下标的含义变了。你的直觉一直是"下标=位置"，突然下标变成了"目标值"，骨架就认不出来了。认出这一点，整类就站稳了。

物品（数组里的每个数）不进下标，而是作为**外层循环**一个一个加进来；真正进状态的是"目标值 j"。

## 1. 大类内部定位：两条正交轴


### 轴 A：每个物品能用几次（管循环方向）

| 取值 | 含义 | 典型题 |
|---|---|---|
| **0/1 背包** | 每个物品最多用一次 | 子集和、目标和 |
| **完全背包** | 每个物品无限次 | 凑零钱、凑零钱 II |
| 多重背包 | 每个物品有限次（先不展开） | —— |

### 轴 B：在"凑出 j"这件事上，你到底求什么（管转移算子）

这就是你那条"决策轴（max/min vs sum）"，只是这一类多了一个 **可行性** 档位：

| 取值 | 转移用什么 | 初始化 `dp[0]` | 其余初值 |
|---|---|---|---|
| **可行性**（能不能凑出）| `dp[j] = dp[j] \|\| dp[j-x]` | `true` | `false` |
| **最值**（最少/最多几个）| `dp[j] = min/max(dp[j], dp[j-x] + 1)` | `0` | `INF` / `-INF` |
| **计数**（有几种凑法）| `dp[j] += dp[j-x]` | `1` | `0` |

### 把你点名的四道题摆进网格

|            | 可行性 (`\|\|`)         | 最值 (`min/max`)   | 计数 (`+`)        |
| ---------- | --------------------- | ----------------- | ---------------- |
| **0/1**    | 分割等和子集 416（子集和） | ——                | 目标和 494         |
| **完全**   | ——                    | 凑零钱 322          | 凑零钱 II 518      |

**这就回答了你最大的困惑：**
子集和 和 凑零钱 **确实是同一大类**，但它们落在网格里**不同的格子**——
- 子集和 416 = (0/1, 可行性)
- 凑零钱 322 = (完全, 最值)

两条轴都不一样，所以手感差很远；但骨架（"dp[j] 挂在目标值上、物品外层枚举"）是同一套。**它们不是两类题，是同一张表的两个角。**

## 2. 最容易写错的：循环顺序 = 两个互相独立的旋钮

清单专门标了这条。它其实由两个**互相独立**的决定组成，分开记就永远不乱：

- **旋钮 ①：内层 `j` 的方向** —— 由轴 A（用几次）决定
- **旋钮 ②：两层循环谁在外** —— 只在轴 B 是"计数"时才有意义（决定算组合还是排列）

下面两个都用**具体例子走一遍**，不抽象。

### 旋钮 ①：`j` 倒序 还是 正序（管"用几次"）

规则：
- **0/1 背包 → `j` 倒序（大 → 小）**
- **完全背包 → `j` 正序（小 → 大）**

**为什么？用一个具体例子看正序会出什么 bug。**

设只有一个物品 `x = 3`，问"能不能恰好凑出 0~6 各个值"，做可行性。初始 `dp[0]=true`，其余 `false`。

❌ 如果**正序**（j 从小到大）：
```
j=3: dp[3] |= dp[0]  → dp[3] = true   （用了一个 3，对）
j=6: dp[6] |= dp[3]  → dp[6] = true   （错！这相当于又用了一次 3，3+3=6）
```
只有一个 3，根本凑不出 6，但正序把它算成了 true —— **物品被用了两次**。

✅ 如果**倒序**（j 从大到小）：
```
j=6: dp[6] |= dp[3]  → dp[3] 此刻还是 false → dp[6] 仍 false  （对）
j=3: dp[3] |= dp[0]  → dp[3] = true                          （对）
```
倒序时，更新 `dp[6]` 用到的 `dp[3]` **还没被这一轮的物品 3 更新过**，所以它代表"还没用 3 时"的状态，物品只被算一次。

> 一句话记牢：**倒序 = 让 `dp[j-x]` 保持"上一轮（还没用当前物品）"的值，所以每个物品只用一次；正序 = `dp[j-x]` 已经是"本轮（用过当前物品）"的值，所以能反复用。**

### 旋钮 ②：物品外层 还是 容量外层（只对计数题有意义）

规则：
- **物品在外、容量在内 → 组合数（无序，每种搭配算一次）** → 凑零钱 II 518
- **容量在外、物品在内 → 排列数（有序，顺序不同算不同）** → 组合总和 IV 377

**为什么？用 coins = [1, 2]、目标 = 3 走一遍。**

人工列举：
- 组合（无序）：`{1,1,1}`、`{1,2}` → **2 种**
- 排列（有序）：`1+1+1`、`1+2`、`2+1` → **3 种**

✅ **物品外层（组合，518 写法）**：
```
dp[0]=1
枚举 coin=1: dp[1]+=dp[0]=1, dp[2]+=dp[1]=1, dp[3]+=dp[2]=1
枚举 coin=2: dp[2]+=dp[0]=2, dp[3]+=dp[1]=2
→ dp[3] = 2   ✅ 对上"组合 2 种"
```
因为 coin=1 整个处理完才轮到 coin=2，`1` 永远排在 `2` 前面，`2+1` 这种"2 在前"的顺序不会被生成 → 自动去重成组合。

✅ **容量外层（排列，377 写法）**：
```
dp[0]=1
j=1: 用1 → dp[1]+=dp[0]=1
j=2: 用1 → dp[2]+=dp[1]=1; 用2 → dp[2]+=dp[0]=2
j=3: 用1 → dp[3]+=dp[2]=2; 用2 → dp[3]+=dp[1]=3
→ dp[3] = 3   ✅ 对上"排列 3 种"
```
因为对每个 j 都把所有硬币重新试一遍，`1+2` 和 `2+1` 会分别在不同顺序里被算到。

> **为什么可行性题、最值题不受嵌套顺序影响？**
> `||` 和 `min/max` 是**幂等 / 可交换**的：先看物品 1 还是先看物品 2，"能不能凑出"和"最少几个"都不会变，也不存在"重复计数"的问题。
> 只有**计数（求和 `+`）**会因为顺序不同而把同一组合数成多份，所以**只有计数题对嵌套顺序敏感**。

### 两个旋钮是独立的，别混

- 旋钮 ①（方向）管的是：**物品用几次**。
- 旋钮 ②（嵌套）管的是：**算不算顺序**（仅计数题）。

举例：凑零钱 II 518 = 完全背包 + 组合计数 →
- 因为是完全 → `j` 正序（旋钮 ①）
- 因为要组合数 → 物品外层（旋钮 ②）
两个旋钮各拨各的。

## 3. 拷问最后一步：把 dp 和方向"推"出来（不靠背）

以 0/1 背包可行性为根，套你熟的"拷问最后一步"。

**二维原始定义**：`dp[i][j]` = 用前 `i` 个物品，能否恰好凑出 `j`。

考虑第 `i` 个物品、要凑出 `j`，**最后一步只有两种可能**：
- **不选第 i 个**：那 `j` 必须已经由**前 i-1 个**凑出 → `dp[i-1][j]`
- **选第 i 个**（需 `j ≥ nums[i]`）：那剩下的 `j - nums[i]` 必须由**前 i-1 个**凑出 → `dp[i-1][j - nums[i]]`
- 满足其一即可：
  ```
  dp[i][j] = dp[i-1][j] || dp[i-1][j - nums[i]]
  ```

注意右边两项**都来自 `i-1`**（上一行）。

**降到一维 `dp[j]`**：我们想把"`i-1` 这一维"省掉，用一个滚动数组复用。关键是：更新 `dp[j]` 时，等号右边的 `dp[j - nums[i]]` **必须仍然代表"前 i-1 个物品"的旧值**（因为公式要的是 `dp[i-1][...]`）。

- 如果 `j` **倒序**更新：处理到 `j` 时，`j - nums[i]`（更小的下标）这一轮**还没被碰过**，它还是上一轮（前 i-1 个）的值 → 正好满足公式 → **0/1 用倒序**。
- 完全背包的公式不一样：选了第 i 个之后，剩下的 `j - x` **还能再选第 i 个**，所以它要的是 `dp[i][j - x]`（本轮、已经考虑过当前物品的值）→ 需要 `j - x` **已经被本轮更新过** → **完全用正序**。

> **结论：倒序 / 正序不是规定，是"公式右边要 i-1 的旧值还是 i 的新值"推出来的。** 记不住方向时，回到这一步推一遍即可。

## 4. 决策速查表（写之前先查这张）

确定 (轴A, 轴B) 之后，下面四样直接定死：

| 轴A | 轴B | `j` 方向 | `dp[0]` | 其余初值 | 转移 |
|---|---|---|---|---|---|
| 0/1 | 可行性 | 倒序 | `true` | `false` | `dp[j] \|= dp[j-x]` |
| 0/1 | 最值(min个数) | 倒序 | `0` | `INF` | `dp[j]=min(dp[j],dp[j-x]+1)` |
| 0/1 | 最值(max价值 v) | 倒序 | `0` | `0`（至多）| `dp[j]=max(dp[j],dp[j-x]+v)` |
| 0/1 | 计数 | 倒序 | `1` | `0` | `dp[j]+=dp[j-x]` |
| 完全 | 最值(min个数) | 正序 | `0` | `INF` | `dp[j]=min(dp[j],dp[j-x]+1)` |
| 完全 | 计数·组合 | 正序 + 物品外层 | `1` | `0` | `dp[j]+=dp[j-x]` |
| 完全 | 计数·排列 | 正序 + 容量外层 | `1` | `0` | `dp[j]+=dp[j-x]` |

**一个容易踩的初始化坑：恰好装满 vs 至多。**
- "恰好凑出 j / 恰好装满"：只有 `dp[0]` 是合法起点，其余初值设成"非法"（可行性用 `false`、求最值用 `±INF`）。
- "容量至多 J 求最大价值"：任何容量从 0 件物品开始都合法（价值 0），所以**全部初始化为 0**。

## 5. 一句话题解（存卡用）

- **分割等和子集 416（子集和）**：sum 为奇数直接 `false`；否则目标 = `sum/2`，做 **0/1 背包可行性**，`dp[j] |= dp[j-num]`，`j` 倒序，`dp[0]=true`。
  → 触发条件："把数组分成两半 / 选一个子集凑出固定和" → 0/1 可行性。

- **目标和 494**：先做**符号分组**——正号集合和 `P`、负号集合和 `N`，由 `P+N=sum`、`P-N=target` 解出 `P=(sum+target)/2`。于是问题变成"选子集凑出 `P` 的方案数" = **0/1 背包计数**（`dp[j]+=dp[j-num]`，倒序，`dp[0]=1`）。前提：`sum+target` 必须**非负且为偶数**，否则 0 种。
  → 它不是新题型，是"把 +/- 翻译成选 / 不选"之后的 416 计数版。

- **凑零钱 322**：**完全背包求最少个数**，`dp[j]=min(dp[j], dp[j-coin]+1)`，`j` 正序，`dp[0]=0` 其余 `INF`；最后 `dp[amount]` 没被更新过（还是 INF）就返回 `-1`。

- **凑零钱 II 518**：**完全背包求组合数**，**物品外层、容量内层、正序**累加 `dp[j]+=dp[j-coin]`，`dp[0]=1`。

- **组合总和 IV 377（对照题）**：和 518 同为"完全 + 计数"，但求**排列数** → **容量外层、物品内层**。放在一起看，专治"组合 vs 排列分不清"。

## 6. 板子库（C++，现场直接抄）

### 6.1 通用骨架（把"求什么"当旋钮填）

```cpp
// ========== 0/1 背包通用骨架（一维滚动）==========
// w[i]: 第 i 个物品占的"目标值"（重量/数值）；J: 目标上限
for (int i = 0; i < n; ++i)
    for (int j = J; j >= w[i]; --j)        // 0/1 → 倒序
        /* 转移旋钮，按下表选一行 */ ;

// ========== 完全背包通用骨架（一维滚动）==========
for (int i = 0; i < n; ++i)
    for (int j = w[i]; j <= J; ++j)        // 完全 → 正序
        /* 转移旋钮 */ ;

// 转移旋钮（把上面的 /* ... */ 换成其中一行）：
//   可行性:        dp[j] = dp[j] || dp[j - w[i]];
//   最少个数:      dp[j] = min(dp[j], dp[j - w[i]] + 1);
//   最大价值(v[i]): dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
//   计数:          dp[j] += dp[j - w[i]];
//
// 初始化旋钮：
//   可行性:   dp.assign(J+1, false); dp[0] = true;
//   最少个数: dp.assign(J+1, INF);   dp[0] = 0;
//   最大价值: dp.assign(J+1, 0);                  // 容量至多
//   计数:     dp.assign(J+1, 0);     dp[0] = 1;
```

> 计数题再叠一层旋钮 ②：**组合 → 物品外层**（如上）；**排列 → 容量外层、物品内层**（见 6.7）。

### 6.2 分割等和子集 416（0/1 · 可行性）

```cpp
bool canPartition(vector<int>& nums) {
    int sum = 0;
    for (int x : nums) sum += x;
    if (sum & 1) return false;                 // 奇数无法均分
    int J = sum / 2;
    vector<char> dp(J + 1, false);             // char 当 bool 用，更省
    dp[0] = true;
    for (int x : nums)
        for (int j = J; j >= x; --j)           // 0/1 → 倒序
            dp[j] = dp[j] || dp[j - x];
    return dp[J];
}
```

### 6.3 目标和 494（0/1 · 计数；先数学变换）

```cpp
int findTargetSumWays(vector<int>& nums, int target) {
    int sum = 0;
    for (int x : nums) sum += x;
    long long s = (long long)sum + target;     // P = (sum + target) / 2
    if (s < 0 || (s & 1)) return 0;            // 非负 + 偶数才有解
    int P = (int)(s / 2);
    vector<long long> dp(P + 1, 0);            // 方案数用 long long 防溢出
    dp[0] = 1;
    for (int x : nums)
        for (int j = P; j >= x; --j)           // 0/1 计数 → 倒序
            dp[j] += dp[j - x];                // 注意：x 可能为 0，倒序下 dp[j]+=dp[j] 即翻倍，正好对应 0 可正可负
    return (int)dp[P];
}
```

### 6.4 凑零钱 322（完全 · 最少个数）

```cpp
int coinChange(vector<int>& coins, int amount) {
    const int INF = amount + 1;                // 哨兵：比任何合法答案都大
    vector<int> dp(amount + 1, INF);
    dp[0] = 0;
    for (int c : coins)
        for (int j = c; j <= amount; ++j)      // 完全 → 正序
            dp[j] = min(dp[j], dp[j - c] + 1);
    return dp[amount] > amount ? -1 : dp[amount];
}
```

### 6.5 凑零钱 II 518（完全 · 组合计数）

```cpp
int change(int amount, vector<int>& coins) {
    vector<long long> dp(amount + 1, 0);
    dp[0] = 1;
    for (int c : coins)                        // 物品外层 → 组合（无序）
        for (int j = c; j <= amount; ++j)      // 完全 → 正序
            dp[j] += dp[j - c];
    return (int)dp[amount];
}
```

### 6.6 经典 0/1 背包求最大价值（板子级模板）

```cpp
// n 个物品，重量 w[i]、价值 v[i]，背包容量 J（至多），求最大总价值
int knapsack01(const vector<int>& w, const vector<int>& v, int J) {
    vector<int> dp(J + 1, 0);                  // "至多" → 全 0
    for (int i = 0; i < (int)w.size(); ++i)
        for (int j = J; j >= w[i]; --j)        // 0/1 → 倒序
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
    return dp[J];
}

// 完全背包求最大价值：只把内层改成正序即可
int knapsackComplete(const vector<int>& w, const vector<int>& v, int J) {
    vector<int> dp(J + 1, 0);
    for (int i = 0; i < (int)w.size(); ++i)
        for (int j = w[i]; j <= J; ++j)        // 完全 → 正序
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
    return dp[J];
}
```

### 6.7 组合总和 IV 377（完全 · 排列计数，和 518 对照）

```cpp
int combinationSum4(vector<int>& nums, int target) {
    vector<unsigned long long> dp(target + 1, 0);  // 排列数增长快，用 unsigned long long
    dp[0] = 1;
    for (int j = 1; j <= target; ++j)          // 容量外层 → 排列（有序）
        for (int x : nums)                     // 物品内层
            if (j >= x) dp[j] += dp[j - x];
    return (int)dp[target];
}
```

> **518 vs 377 放一起记**：完全背包 + 计数，唯一区别是谁在外层。
> 物品外 = 组合（518）；容量外 = 排列（377）。这就是旋钮 ② 的全部内容。

## 7. 自检清单（写完扫一遍）

1. 我这题在网格哪个格子？（轴A：用几次；轴B：求什么）
2. 方向对吗？0/1 倒序 / 完全正序。
3. 是计数题吗？是的话——要组合还是排列？组合=物品外层，排列=容量外层。
4. 初始化对吗？可行性 `dp[0]=true`；最值 `dp[0]=0` 其余 INF；计数 `dp[0]=1`。
5. "恰好" 还是 "至多"？至多就全 0，恰好就只有 `dp[0]` 合法、其余设非法值。
6. 方案数会不会溢出？会就上 `long long` / `unsigned long long`。
