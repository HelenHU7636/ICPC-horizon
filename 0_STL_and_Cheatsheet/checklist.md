# 🩺 Data Flow Check (数据流检查单)

当你毫无头绪、不知道代码哪里出 Bug 的时候，强迫自己按照以下步骤对核心变量进行“审查”：

## 变量的“一生” (Lifecycle)
对代码里的每一个关键变量，逐一灵魂拷问：
1. 它在哪里被**创建 (Created)**？
2. 它在哪里被**获取/赋予初始值 (Initialized)**？
3. 它在哪里被**使用 (Used)**？
*👉 缺任意一环，必报错！*

## 两大高频死穴
* **未初始化比较**：在变量还没有获得初始值的时候，绝对不能进行需要初始值作为前提的“比较”类操作。
* **循环状态审查**：对每一个循环体，在纸上问自己：**这一轮循环结束时，这个变量（如 `name`, `grade`）到底应该是什么值？**
