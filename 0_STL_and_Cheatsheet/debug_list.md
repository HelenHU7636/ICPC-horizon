# 极限测试用例 (Test Cases) 构造指南

**核心心法**：测试的时候，绝对不要想“我随便测几个正常输入”。你要想的是：**“如果我的代码错了，它最可能错在哪里？我要专门设计样例把它炸出来。”**

对于枚举、模拟等各类算法题，请严格按照以下 8 个维度来构造你的“炸弹样例”：

## 一、 边界轰炸 (Boundary Bombing)
代码最容易死在 `for (int i = 0; i < n - 2; i++)` 这种边界控制上。重点怀疑：循环边界有没有漏？

* **1. 最小合法数据范围**
  * **目的**：测试极小输入下是否会越界或输出乱码。
  * **举例**：题目要求选 3 个数，那就只给 3 个数 `[1, 2, 3]`。这种样例一眼就能看出答案，无需手算。
* **2. 答案在最前面 / 最后面**
  * **目的**：专查 `i, j, k` 的循环有没有漏掉第一个或最后一个元素。
  * **举例 (三数之和)**：
    * 答案在最前：`[1, 2, 3, 100, 200]`，目标是 6。
    * 答案在最后：`[1, 2, 3, 100, 200]`，目标是 303。

## 二、 逻辑精准度测试 (Logic Precision)
重点怀疑：有没有把不合法情况算进去？更新答案的逻辑是否严密？

* **3. 刚好命中目标 (Exact Match)**
  * **目的**：检查代码在“差距为 0”时，能否正确捕捉并更新答案。
  * **举例**：数组 `[8, 3, 7, 10, -2, 5]`，目标 `20`。预期输出就是 `20`（因为 8+7+5=20）。
* **4. 无法命中，只能选最接近 (Closest Match)**
  * **目的**：测试“求最小绝对差”的逻辑。
  * **举例**：数组 `[1, 2, 4, 8, 16]`，目标 `10`。手算最近组合是 `1+2+8=11`。
* **5. 平局情况 (Tie-Break)**
  * **目的**：题目要求“差距相同，输出较小值”时的仲裁机制。
  * **举例**：数组 `[1, 2, 6, 8, 20]`，目标 `10`。组合有 `1+2+6=9` (差1) 和 `1+2+8=11` (差1)。代码必须输出 `9`。

## 三、 特殊值陷阱 (Special Values)
重点怀疑：值相同和位置相同的概念有没有混淆？负数处理逻辑对不对？

* **6. 包含负数 / 0 / 重复数字**
  * **目的**：全正数能过的代码，碰负数极容易错。同时测试代码是否能正确处理“位置不同但数值相同”的情况。
  * **举例**：
    * 负数测试：`[-5, -2, 1, 3, 4, 10]`，目标 `0`。
    * 重复数测试：`[2, 2, 2, 2, 2]`，选 3 个数目标 `6`。题目要的是“不同位置的队员”，而不是“不同数值”，所以答案必须是 `6`。

## 四、 系统压力测试 (Stress Testing)
重点怀疑：会不会跑太慢 (TLE)？内存会不会爆 (MLE)？变量类型有没有溢出？

* **7. 极端大数 (检查整数溢出)**
  * **目的**：强迫代码暴露 `int` 溢出问题。
  * **举例**：三个数相加，如果单个数字高达 $10^9$，加起来必定爆 `int`。**铁律：只要涉及求和、乘法、答案极大，无脑上 `long long`。**
* **8. 最大规模测试时间 (TLE 测试)**
  * **目的**：验证算法复杂度。不需要手算答案，只看会不会卡死。
  * **举例**：题目给出 $N \le 500$，如果写了三层循环，计算量约为 $500^3 = 1.25 \times 10^8$，处于超时的边缘。丢一个 $N=500$ 的全 1 数组进去跑一下试试时间。

## 🛠️ 考场实战 SOP：10 步验收清单
每次写完代码，提交 (Submit) 之前，深呼吸，强迫自己按这个清单走一遍测试：

- [1.]  最小样例 (Minimum input)
- [2.]  普通样例 (Normal case)
- [3.]  答案在开头 (Answer at head)
- [4.]  答案在结尾 (Answer at tail)
- [5.]  刚好命中 (Exact match)
- [6.]  没有命中 (Closest match)
- [7.]  平局仲裁 (Tie-break)
- [8.]  负数 / 0 / 重复数 (Negatives/Zeros/Duplicates)
- [9.]  极大值防溢出 (Max values for `long long`)
- [10.]  最大 $N$ 压力测试 (Max $N$ for TLE)

一套完整的轰炸走下来如果都防住了，这题 99% 就能 AC (Accepted)！
