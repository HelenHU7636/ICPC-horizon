## 1. Vector (动态数组)
**核心逻辑**：内存连续，支持随机访问。尾部插入极快，中间插入极慢。
**⚠️ 避坑笔记**：必须要有初始大小（Size）才能通过索引 `[]` 赋值，否则会引发 Segmentation Fault。

* **初始化与基础**
  * `vector<int> v;` // 创建空 vector
  * `vector<int> v(10, 0);` // 创建大小为 10，全初始化为 0 的 vector
* **增删改查**
  * `v.push_back(x);` // 在尾部插入元素 x (O(1))
  * `v.pop_back();` // 删除尾部元素 (O(1))
  * `v.insert(v.begin() + i, x);` // 在第 i 个位置插入 x (O(N) 极慢，慎用)
* **容量与状态**
  * `v.size();` // 返回当前元素个数
  * `v.empty();` // 判断是否为空，返回 bool
  * `v.clear();` // 清空所有元素
  * `v.resize(n);` // 重新指定大小为 n

## 2. Queue (队列)
**核心逻辑**：先进先出 (FIFO)。用于处理“谁先发现，谁先处理”的问题。
**🚀 笔记高光**：BFS 最短路径问题的核心结构，“水波纹式扩展”，距离近的状态一定先于远的状态被扩展。

* **核心操作** (注意：队列不能使用迭代器，不能随机访问)
  * `queue<int> q;` 
  * `q.push(x);` // 新元素进入等待区（排到队尾）
  * `q.pop();` // 弹出队头元素（当前处理完了），**无返回值**！
  * `q.front();` // 查看队头元素是谁（当前要处理的）
  * `q.back();` // 查看队尾元素是谁
  * `q.empty();` // 检查等待区还有没有任务
  * `q.size();` // 队列中元素的个数

## 3. Stack (栈)
**核心逻辑**：后进先出 (LIFO)。
**🚀 笔记高光**：叠盘子模型，处理“撤销”动作（最近的动作最先被撤销），以及括号匹配、嵌套结构处理。

* **核心操作** (只能操作最顶端元素，不能随机访问)
  * `stack<int> st;`
  * `st.push(x);` // 把一个东西压到最上面
  * `st.pop();` // 弹出最上面的元素，**无返回值**！
  * `st.top();` // 查看最上面的元素是谁（不会删除）
  * `st.empty();` // 判断是否为空
  * `st.size();` // 返回当前栈内元素个数

## 4. Set (集合)
**核心逻辑**：底层为红黑树。内部元素**自动排序**，且**绝对不重复**。操作复杂度为 O(log N)。

* **核心操作**
  * `set<int> s;`
  * `s.insert(x);` // 插入元素 x，如果已存在则什么都不做
  * `s.erase(x);` // 删除值为 x 的元素
  * `s.count(x);` // 判断 x 在不在。由于 Set 元素唯一，只返回 0 或 1
  * `s.find(x);` // 查找 x，返回指向 x 的迭代器 (Iterator)。如果找不到，返回 `s.end()`
* **进阶遍历**
  * `for (auto it = s.begin(); it != s.end(); ++it) { cout << *it; }`

## 5. Map (映射 / 字典)
**核心逻辑**：底层为红黑树。存储键值对 (Key-Value)，根据 Key **自动排序**。操作复杂度为 O(log N)。

* **核心操作与避坑**
  * `map<string, int> mp;`
  * `mp["abc"]++;` // **🚀 笔记高光**：极其方便的计数器！如果 "abc" 不存在，会自动创建对应的 Key，Value 默认为 0，然后再 ++。无需分类讨论。
  * `mp.insert({key, value});` // 标准插入方式
  * `mp.erase(key);` // 删除键为 key 的键值对
* **查询操作**
  * `mp.count(key);` // 只问有没有这个 Key，返回 0 或 1。**不会**偷偷修改 Map。
  * `mp.find(key);` // 找这个 Key 的位置，返回迭代器。找不到就返回 `mp.end()`。
* **⚠️ 致命陷阱**
  * `if (mp["apple"] == 1)`：即使你只是想“查”一下 "apple"，但用 `[]` 访问时，如果 "apple" 不存在，Map **会偷偷给你创建一个** `{"apple", 0}`。所以查询一定要用 `count()` 或 `find()`！
